# ТЗ на написание тестов

## 1. Цель

Проверить корректность **чистой логики** (`main/board.lua`, `main/match.lua`) без Defold GUI: ходы, победа/ничья в партии, счёт матча до **2 побед**, чередование первого хода, фазы и ответ `try_move`.

## 2. Объект тестирования

| Компонент | Что покрывать |
|-----------|----------------|
| **board** | Валидность хода, победитель по линиям, ничья, отказ по индексу и занятой клетке |
| **match** | Жизненный цикл матча, `try_move`, `start_next_round`, `reset_match`, переходы `phase` |

GUI и ввод не входят в автотесты.

## 3. Среда

- **Lua 5.1**, запуск из корня проекта: `lua test/run_tests.lua`
- Точка входа подключает `test/test_board.lua` и `test/test_match.lua` через `require`

## 4. Кейсы для `board`

1. Пустая доска — `winner` = `nil`
2. Победа по строке / столбцу / диагонали — возвращается верный игрок `1` или `2`
3. Заполненная доска без линии — `is_draw`, `winner` = `nil`
4. Невалидный индекс — `apply_move` отклоняется (`bad_index`)
5. Занятая клетка — `occupied`
6. Корректный ход — клетка заполняется значением игрока

## 5. Кейсы для `match`

1. Неверная фаза — `try_move` в `between_rounds` даёт `wrong_phase`
2. Занятая клетка — `occupied`
3. Чередование ходов в партии
4. Победа в партии без завершения матча — `phase == between_rounds`, счёт +1
5. `start_next_round` — смена `starting_player`, инкремент `round`, пустая доска, `playing_round`
6. Матч **2:0** — после второй победы одного игрока `match_over`, повторный `try_move` с `match_over`
7. Ничья в партии — счёт не меняется, `round_draw`, `between_rounds`

## 6. Критерии приёмки

- Все группы кейсов выше покрыты тестами
- `lua test/run_tests.lua` завершается с кодом 0 и строкой `All tests passed.`
