# ТЗ на написание кода (программист)

## 1. Продукт

Игра «Крестики-нолики» в Defold: матч **до двух побед в партиях** (best-of-3). Ничья в партии — без очков, новая партия. В каждой новой партии **первым ходит другой игрок**. После второй победы у одного игрока — окончание матча и финальный попап; отдельный попап после **ничьей в партии** (перед следующей партией).

## 2. Архитектура

- **`main/board.lua`** — только доска: ход, победитель, ничья, валидация индекса
- **`main/match.lua`** — состояние матча, `try_move`, `start_next_round`, `reset_match`, вызовы `board`
- **`main/game.gui_script`** — маппинг узлов `cell1`…`cell9`, отображение X/O, HUD, попапы; в обработчике ввода **нет** дублирования правил линий — только вызов API матча и обновление вида по результату

## 3. Состояние матча

- `scores[1]`, `scores[2]` — число выигранных партий (0…2)
- `round` — номер текущей партии (может расти при серии ничьих)
- `starting_player`, `current_player`
- `board` — 9 клеток, значения `nil`, `1`, `2`
- `phase`: `playing_round` | `between_rounds` | `match_over`

## 4. Контракт `try_move`

Возвращаемая таблица (минимум):

- `ok`, `reason` при отказе
- `round_ended`, `round_draw`, `winner_round` (если не ничья)
- `match_ended`, `winner_match`

При `match_over` дальнейшие ходы отклоняются.

## 5. GUI

- Клик по клетке в `playing_round` — `try_move`
- Конец партии — попап (ничья или победа в раунде), по тапу — `start_next_round` (если матч не окончен)
- Конец матча — попап победителя (текст «священника» как сюжетный элемент), по тапу — `reset_match`

## 6. Критерии приёмки

- Соблюдено разделение логики и GUI
- Поведение соответствует `docs/TZ_TESTS.md` (тесты проходят)
- Проект открывается в Defold, коллекция `main/main.collection` запускает игру
